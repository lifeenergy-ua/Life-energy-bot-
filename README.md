# Life Energy — Telegram-бот першого контакту

Бот бере на себе розмову з лідом від першого повідомлення до моменту, коли
людина готова до безкоштовної діагностики. Далі — все живе спілкування веде
Ірина або партнер. Бот нічого не продає і не називає фінальну ціну сам.

## Що робить бот

1. Вітає і питає, що цікавить (фінансова грамотність / дохід / професія /
   інвестиції / бізнес / "не знаю").
2. Дає відповідну відповідь зі скрипту адміністратора Life Energy.
3. Питає, коли зручно поговорити, і бронює безкоштовну діагностику.
4. Питає ім'я та телефон, записує ліда в Google Таблицю (вашу CRM).
5. Одразу надсилає повідомлення адміну (вам) у Telegram з даними ліда.
6. На слово «СТАРТ» в будь-який момент — перезапускає сценарій напряму.
7. На запитання про ціну чи "що це таке" — відповідає завченим текстом.
8. Якщо не розуміє повідомлення — не вигадує відповідь, а одразу пересилає
   його вам і повідомляє людині, що консультант зв'яжеться особисто.
9. Якщо людина обрала "цього тижня" і мовчить — через 24 години бот сам
   нагадує про себе.

## Крок 1. Отримати токен бота

1. Напишіть у Telegram боту **@BotFather** команду `/newbot`.
2. Дайте боту ім'я та юзернейм (наприклад `LifeEnergy_bot`).
3. BotFather надішле токен виду `123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` —
   збережіть його, це `BOT_TOKEN`.

## Крок 2. Дати доступ до Google Таблиці

1. Зайдіть у [Google Cloud Console](https://console.cloud.google.com/).
2. Створіть проєкт → увімкніть **Google Sheets API** та **Google Drive API**.
3. Створіть **Service Account** (IAM & Admin → Service Accounts → Create).
4. Створіть ключ типу JSON для цього акаунта — файл завантажиться на
   комп'ютер. Перейменуйте його на `google_credentials.json`.
5. Відкрийте ваш email сервісного акаунта (виглядає як
   `xxx@xxx.iam.gserviceaccount.com`, є у файлі ключа) і **надайте йому
   доступ редактора** до вашої Google Таблиці (кнопка "Поділитися").
6. ID таблиці вже прописаний у коді (`sheets.py`) — це той, що ви дали:
   `1C8qKB-fpGvoWQyA89pAvtNIlhizuIeyRJUSLv3zKp6A`.
7. У таблиці створіть (або перейменуйте) вкладку з назвою `Leads` — бот сам
   допише туди заголовки колонок при першому запуску, якщо їх ще немає.

## Крок 3. Дізнатись свій ADMIN_CHAT_ID

1. Запустіть бота локально (крок 4) без `ADMIN_CHAT_ID`.
2. У Telegram напишіть боту `/whoami` — він відповість вашим chat_id.
3. Додайте це значення як `ADMIN_CHAT_ID` (див. крок 4) і перезапустіть бота.

## Крок 4. Запуск

```bash
cd life_energy_bot
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt

export BOT_TOKEN="ваш_токен_з_botfather"
export ADMIN_CHAT_ID="ваш_chat_id"          # можна залишити порожнім на першому запуску
export GOOGLE_CREDENTIALS_PATH="google_credentials.json"

python bot.py
```

Покладіть `google_credentials.json` у ту саму папку, що й `bot.py`.

## Крок 5. Безкоштовний хостинг (щоб бот працював 24/7)

Локально бот працює, лише поки увімкнений ваш комп'ютер. Для постійної
роботи — безкоштовні варіанти:

- **Railway.app** — підключаєте GitHub-репозиторій з цією папкою, задаєте
  ті самі змінні середовища (`BOT_TOKEN`, `ADMIN_CHAT_ID`,
  `GOOGLE_CREDENTIALS_PATH` — файл ключа завантажте як secret file), Railway
  сам запускає `python bot.py`. Є безкоштовний ліміт годин на місяць.
- **Render.com** — аналогічно, "Background Worker", безкоштовний план з
  обмеженнями.
- **PythonAnywhere** — простіше для новачків, є безкоштовний тариф.

Я можу підготувати конкретні файли конфігурації (`Procfile`, `railway.json`
тощо) під той сервіс, який оберете.

## Що ще налаштувати вручну

- `ADMIN_CHAT_ID` — обов'язково, інакше повідомлення про нових лідів нікуди
  не прийдуть.
- Якщо захочете, щоб бот повідомляв не тільки вас, а й конкретного партнера
  залежно від теми (наприклад, страхування → Наталія Стороженко) — це легко
  додати окремим правилом, скажіть, і я допишу логіку.
